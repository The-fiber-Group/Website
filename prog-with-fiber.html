Basic Syntax

Tokens

Fiber syntax is made of two kinds of tokens, symbols and terminals. Symbols are used for keywords and naming things like variables or functions. Terminals are used for operators and delimiters. The two of these together are used to make various expressions.

Symbols are made of alphanumeric characters and the dash character. Terminals are made of non-alphanumeric characters including the dash character. Because both symbols and terminals can use the dash character, symbols cannot begin or end with a dash.

It's important to remember that symbols are case-insensitive!

This is a symbol: a----sym-bol
This is the same symbol: A----SYM-BOL

This is not a symbol: --symbol--

All tokens are delimited by whitespace, blank characters, and other non-printable characters such as line feeds and tabs. There are no special characters or delimiting tokens. Additionally, symbols can delimit terminals and terminals can delimit symbols. These rules produce syntax that is slightly different than normal programming languages. For example:

This is a pair of nested parentheses: ( ( ) )

This is a single pair of double-paren delimiters: (( ))

In the two expressions, x*- and x* -, the asterisk and dash form three different operators, *- from the first expression, * and - from the second due to their being separated by a space.

This behavior may seem harder to read at first, but the requirement of verbose spacing between separate tokens reduces terse expressions that would normally be ambiguous in other languages.

Numbers

Numbers can be either decimal or hexadecimal. Decimal numbers can use a sign prefix and a radix point. Hexadecimal numbers use the digits 0-9 and the capital letters A-F and they have to be prefixed with "0x".

Here are some examples of numbers:
1
1.1
-0.010
0xDEADBEEF

Strings

Strings can contain ASCII characters, Unicode characters, or raw binary data as a "byte string". All strings are surrounded by double quotations. Byte strings replace the opening double quotation with a special opening quotation, #".

Strings are ASCII by default, unless they include a Unicode character via a UTF-8 encoded character in the source file or through a character escape sequence in the string. Single characters are escaped into a string using the backslash prefix. Double quotations and backslashes are included in a string like so: "\"", "\\"

Non-printable characters are inserted using their hexadecimal encoded number. The space character would be escaped into a string using "\20". Unicode characters are escaped by adding a capital U after the backslash and then a four digit hexadecimal number, "\U20F0".

Byte strings use a series of two-digit hexadecimal codes to specify each 8 bit byte value in the string. Spaces are allows, but not required between each number in a byte string. Ex: #"01 FF A3" is the same as #"01FFA3"

Character Literals

Character literals are represented simply by including only one character inside a string, "A".

Data Types

Data types are declared by surrounding the data type definition in a closed pair of brackets, [ ... ].
